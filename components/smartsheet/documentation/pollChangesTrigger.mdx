The **New and Updated Sheets** polling trigger checks Smartsheet on a configured schedule for sheets that were created or modified since the last poll. Unlike webhook-based triggers, polling does not require webhook subscription management and works reliably for low-frequency monitoring of sheet inventory changes.

#### How It Works

On each scheduled execution, the trigger:

1. Reads the last poll timestamp from the polling state (or uses the current execution time on the first run) and formats it as a Smartsheet-compatible timestamp
2. Calls the Smartsheet `/sheets` endpoint with a server-side `modifiedSince` filter, walking all paginated results
3. Partitions the returned sheets into **created** and **updated** buckets by comparing each sheet's `createdAt` value against the last poll cutoff: sheets whose `createdAt` is after the cutoff land in the `created` bucket, and all other returned sheets land in the `updated` bucket
4. Persists the current execution time as the new last poll timestamp for the next run
5. Emits the partitioned results in the trigger payload, filtered by the **Show New Records** and **Show Updated Records** toggles

#### Returned Data

The trigger emits a payload with two arrays, `created` and `updated`, each containing sheet summaries from Smartsheet.
