The [BambooHR](https://www.bamboohr.com/) **Changed Employees** polling trigger checks for new, updated, and deleted employees on a configured schedule. It calls the `GET /v1/employees/changed` endpoint and partitions the response into newly inserted employees and updated or deleted employees.

For details on the underlying endpoint, refer to the [Get Last Changed Info reference](https://documentation.bamboohr.com/reference/get-last-changed-info).

#### How It Works

On each scheduled execution, this trigger:

1. Reads the `lastPolledAt` timestamp from cross-flow polling state (defaults to the current time on the first run, so the first poll returns no changes).
2. Calls the BambooHR API for employees changed since `lastPolledAt`.
3. Partitions the response into `created` (records with `action: "Inserted"`) and `updated` (records with `action: "Updated"` or `"Deleted"`).
4. Persists the current timestamp as the new `lastPolledAt` for the next run.
5. Returns the filtered results. When both **Show New Employees** and **Show Updated Employees** are disabled, the API call is skipped entirely.
